# Affiliate Content Engine — Project Overview

## מטרת הפרויקט

Affiliate Content Engine היא מערכת פנימית שמטרתה לאתר מוצרים מאתרי Affiliate, להתחיל עם AliExpress, לשמור אותם במסד נתונים, ולאפשר פרסום מהיר שלהם לערוץ טלגרם.

בשלב הראשון המטרה היא לבנות MVP קטן, פשוט ויציב שעובד מקצה לקצה.

---

## זרימת MVP ראשונית

ה-MVP הראשון צריך להוכיח זרימה בסיסית אחת:

AliExpress → Product → Supabase → Products Page → Telegram

Gemini / AI Content יהיה אופציונלי ולא יהווה חסם לפרסום.

---

## מה המערכת צריכה לעשות ב-MVP

המערכת צריכה לאפשר:

1. סריקת מוצרים מ-AliExpress לפי קטגוריה.
2. שמירת מוצרים ב-Supabase.
3. הצגת המוצרים במסך Products Page.
4. פרסום מוצר בסיסי לטלגרם ללא AI.
5. יצירת תוכן שיווקי בעברית בעזרת Gemini כאפשרות אופציונלית.
6. פרסום מוצר לטלגרם עם טקסט AI, אם קיים.

---

## מה זה Supabase

Supabase הוא מסד נתונים בענן.

בפרויקט הזה הוא משמש לשמירת:

* מוצרים שנסרקו.
* קישורי Affiliate.
* סטטוס מוצר.
* היסטוריית פרסומים.
* הגדרות בסיסיות.

המטרה היא שהמערכת תעבוד גם כשהמחשב המקומי כבוי, ותוכל בעתיד לעלות ל-Vercel בלי לאבד נתונים.

---

## למה צריך Products Page

Products Page הוא מסך פשוט שבו רואים את המוצרים שהמערכת מצאה.

המסך נדרש כדי:

* לוודא שהסריקה עובדת.
* לראות את המוצר לפני פרסום.
* לבדוק תמונה, מחיר וקישור.
* לפרסם ידנית מוצר אחד לטלגרם.
* להפעיל AI רק כשצריך.

זה לא דשבורד מתקדם ולא מערכת אישורים מלאה. זה מסך עבודה בסיסי ל-MVP.

## עקרון העבודה המרכזי

קודם בונים מערכת קטנה שעובדת.

רק אחרי שיש זרימה מלאה שעובדת מקצה לקצה, מוסיפים שכבות נוספות.

אין להוסיף פיצ׳רים חדשים לפני שה-MVP יציב.

---

## ארכיטקטורת MVP

```text
AliExpress API
      ↓
Scan Service
      ↓
Supabase Database
      ↓
Products Page
      ↓
Basic Telegram Publisher
      ↓
Optional AI Content Generator
```

---

## מקורות מידע חיצוניים

ב-MVP משתמשים ב:

* AliExpress Official Affiliate API
* Telegram Bot API
* Supabase PostgreSQL
* Gemini API כאופציונלי

---

## Stack טכני

המערכת תיבנה עם:

* Next.js
* TypeScript
* Supabase PostgreSQL
* Prisma
* Telegram Bot API
* AliExpress Official Affiliate API
* Gemini API כאופציונלי

Next.js נבחר כי הפרויקט דורש גם Frontend וגם Backend API Routes באותו פרויקט.

## הגדרת הצלחה ל-MVP

ה-MVP נחשב עובד רק כאשר ניתן לבצע את הפעולות הבאות:

1. ללחוץ על כפתור Scan.
2. לקבל מוצרים אמיתיים מ-AliExpress.
3. לשמור אותם ב-Supabase.
4. לראות אותם במסך Products Page.
5. לבחור מוצר אחד.
6. לפרסם אותו לטלגרם בלי AI.
7. אופציונלית — ליצור לו טקסט בעברית עם Gemini.
8. לפרסם אותו לטלגרם עם טקסט AI.

אם שלבים 1-6 לא עובדים — ה-MVP עדיין לא מוכן.
